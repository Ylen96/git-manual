Git Commit

Description

Syntaxe :

Syntaxe d'utilisation de la commande
Exemple d'affichage dans le terminal

git commit -m

Cette commande va enregistrer les modifications préalablement faites avec la commande git add directement dans l'historique et va permettre de leur associer un nom, ou message descriptif

Syntaxe :

$git commit "message descriptif"
git add fichier1.html
git add fichier2.html
git commit -m "liens CSS"
[gh-pages cbd590d] liens CSS
2 files changed, 2 insertions (+), 2 deletions (-)

git commit -a

Cette commande permet de faire un commit non précédé de la commande git add, ainsi les fichiers modifiés entre deux commit peuvent directement être enregistré dans l'historique

Syntaxe :

$git commit -am "message descriptif"
On branch gh-pages
Your branch is up-to-date with 'origin/gh-pages'.

modified : fichier1.html

no changes added to commit (use "git add" ans/or "git commit -a")
git commit -am "mise a jour fichier 1"

git commit --amend

La commande git commit --amend permet de faire des modifications sur les métadonnées du dernier commit si vous avez fait des erreurs dans le commentaire ou si vous avez oublié de add un ficher.
ATENTION: Cela change l'histoire (et le SHA) donc n'utilisez pas cette dernière si vous avez déja publié vos commits!

Syntaxe :

$git commit --amend -m "modification de votre message de commit"
git commit --amend -m "modification de votre message de commit"
[gh-pages 512fd5c] modification de votre message de commit
1 file changed, 1 insertion(+),1 deletion(-)
