Git Blame

Cette commande va afficher les lignes d'un fichier et leur associer le dernier commit qui les concernes, afin de pouvoir comparer chaque ligne, et trouver d'eventuels problèmes.

Syntaxe :

$git blame nom_du_fichier
$ git blame exemple.html
a153f89d (Ylen96 2015-03-12 15:07:46 +0100 1) dossier X, Y et Z
1501d1d9 (Ylen96 2015-03-12 15:12:23 +0100 2) fichier exemple
a153f89d (Ylen96 2015-03-12 15:07:46 +0100 3)
1501d1d9 (Ylen96 2015-03-12 15:12:23 +0100 4) 0782592389

git blame -L

Cette commande va permettre de restreindre les lignes affichées par le ""blame"" et donc de diminuer le nombre de ligne visibles, et à comparer.
Pour annoter les lignes que l'on souhaite observer on va utiliser, "-L " pour le début, ainsi on peut observer les lignes depuis le début choisi jusqu'à la fin du fichier

Syntaxe :

$git blame -L nom_du_fichier
$ git blame -L 45 branch.html 57221160 (Melanie13340 2015-03-08 17:35:30 +0100 45) <"/footer">
57221160 (Melanie13340 2015-03-08 17:35:30 +0100 46)
57221160 (Melanie13340 2015-03-08 17:35:30 +0100 47) <"/body">

git blame --reverse

Au lieu d'afficher les lignes et de leur associer le dernier commit qui les à modifiées, cette commande va nous permettre d'observer les lignes modifier à partir d'un SHA (commit). Ceci exige une gamme de révision le DÉBUT .. LA FIN où le chemin pour blame existe dans le DÉBUT.

Syntaxe :

git blame --reverse nom_du_fichier
